# UDP 3-Nodes VANET Example

A simple VANET simulation with 3 vehicles communicating over Wi-Fi using UDP,
with mobility generated by SUMO and imported into NS-3.

## Scenario

```
[ v0 - sender ] ---wifi---> [ v1 - relay ] ---wifi---> [ v2 - receiver ]
```

- **v0 (sender):** sends UDP packets to v2
- **v1 (relay):** intermediate node, forwards packets via OLSR routing
- **v2 (receiver):** UDP echo server, receives packets from v0

The nodes move according to a mobility trace exported from SUMO.
OLSR (Optimized Link State Routing) is used as the routing protocol.

## Project Structure

```
examples/
├── sumo/
│   ├── map.nod.xml       # nodes (intersections)
│   ├── map.edg.xml       # edges (streets)
│   ├── map.net.xml       # compiled network
│   ├── veh.rou.xml       # vehicle routes
│   ├── sim.sumocfg       # SUMO main config
│   ├── fcd.xml           # floating car data (simulation output)
│   ├── mobility.tcl      # NS-3 mobility trace (exported from SUMO)
│   └── generate.py       # script to generate all SUMO files and export to NS-3
└── ns3/
    └── vanet-simple.cc   # NS-3 simulation script
```

---

## Step 1: Generate SUMO mobility and export to NS-3

```bash
cd examples/sumo/
python3 generate.py
```

This script will:
1. Build the road network from `map.nod.xml` and `map.edg.xml`
2. Generate the SUMO configuration file
3. Run the SUMO simulation and produce `fcd.xml`
4. Export the mobility trace to `mobility.tcl` (NS-3 format)

To delete previously generated files and start fresh:

```bash
python3 generate.py --clean
```

---

## Step 2: Run the NS-3 simulation

```bash
cd ns-3-dev/
./ns3 run scratch/vanet-simple
```

## NS-3 Visualization with NetAnim

### Installing NetAnim

```bash
cd ~/HelloWorld_SUMONS3/

git clone https://gitlab.com/nsnam/netanim.git

cd netanim

mkdir build && cd build

cmake .. && cmake --build .

make -j$(nproc)
```

### Running NetAnim

```bash
./netanim
```

Open the `animation.xml` file generated by the simulation to visualize the scenario.
